# コーディングスタイル

この本は初心者エンジニア向けの入門書。認知負荷を下げ、理解しやすいコードを心がける。

## コメント
- **1行の日本語コメント**を適度な間隔で追加
- 複数行コメントは避ける

## 横幅制限（重要）
- 書籍のDTPでは**半角70文字**を超えると折り返しが発生する
- コードは70文字以内に収まるよう工夫する（変数名の短縮、引数の改行、文字列の分割など）
- 70文字に収まらない場合は、読みやすい位置であらかじめ改行しておく
- ただし行数が多くなりすぎないよう、改行と簡潔さのバランスを取る
- **バックスラッシュ（`\`）による行継続は見づらいので使わない**

## コード構造
- **関数化せずベタ書き**を基本とする（処理の流れが追いやすい）
- エラーハンドリング等で冗長にならないよう**シンプルに**
- 抽象化・共通化より**わかりやすさ優先**

## 命名規則
- 変数名は**短すぎず意味がわかる**長さに（`r` より `result`、`resp` より `response`）

## 例

```python
# クライアントを作成
client = boto3.client("bedrock-agent-runtime")

# メモリストアを取得
response = client.get_memory(memoryId="my-memory-store")

# メモリの内容を表示
print(response["memoryContents"])
```
